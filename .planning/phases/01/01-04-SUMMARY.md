## Decisions Made
- **Path-based layout routes for customer/agent:** Plan specified `_customerAuthenticated` and `_agentAuthenticated` (pathless layout routes), but TanStack Router requires unique path roots per layout. Pathless routes with index pages all resolve to `/` causing conflicts. Used `customer/` and `agent/` path-based routes instead, which aligns better with URL structure and avoids conflicts.
- **Redirect targets use portal roots:** Login success and authenticated-redirect targets use `/customer` and `/agent` instead of `/customer/dashboard` and `/agent/dashboard` since child routes don't exist yet. Future phases adding dashboard pages under these paths will work seamlessly.
- **401 handler removed from queryCache:** The old handler that called `useAuthStore.getState().adminClearAuth()` and redirected with search params was the template's mock pattern. Auth error handling is now done by Axios interceptors (created in 01-02).
