    **2. Create `website/src/hooks/use-ping.ts`**
    - Custom hook managing WebSocket connection for ping per D-04 and UI-SPEC "Real-Time Ping Display"
    - Parameters: takes routerId (string | null)
    - Returns: `{ latency: string, status: 'connecting' | 'connected' | 'error' | 'unavailable' }`
    - Behavior:
      - When routerId is null: return { latency: '-- ms', status: 'unavailable' } -- no WS connection attempt
      - When routerId changes: close existing WebSocket, open new one to `ws://${window.location.host}/api/v1/routers/${routerId}/monitor/ws/ping?address=8.8.8.8`
      - WebSocket message handling: parse JSON message, extract latency/ttl fields, update latency state
      - Error/close: set status to 'error', latency to 'timeout'
      - Open: set status to 'connected'
      - Cleanup: close WebSocket on unmount and when routerId changes
      - Reconnection: on close (not unmount), attempt reconnect after 3 seconds, max 5 attempts
    - Auth: Read adminAccessToken from useAuthStore.getState().adminAccessToken. Pass token as query param: `&token=${token}`
    - Use useRef for WebSocket instance to avoid stale closures
    - Use useCallback for message handler
  </action>
