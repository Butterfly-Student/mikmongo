## Decisions Made

- Shadcn Nova preset selected (Lucide icons, Geist font) over the plan's "New York / Neutral" — Nova is the default for Tailwind v4 shadcn init and includes better sidebar tokens
- tsconfig.json duplicates paths from tsconfig.app.json — shadcn CLI reads the root tsconfig, not referenced tsconfigs
- `shadcn init --preset nova --yes` used for non-interactive init (interactive mode hangs in CI/scripted environments)

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Added path alias to root tsconfig.json for shadcn CLI detection**
- **Found during:** Task 6 (Shadcn init)
- **Issue:** `npx shadcn@latest init` failed with "No import alias found in tsconfig.json" — the CLI reads the root tsconfig.json directly and does not traverse `references` to tsconfig.app.json
- **Fix:** Added `compilerOptions.paths` to root tsconfig.json (kept references array intact)
- **Files modified:** dashboard/tsconfig.json
- **Verification:** shadcn init succeeded with "Validating import alias. OK"
- **Committed in:** 94e669f (Task 6 commit)

**2. [Rule 3 - Blocking] Created globals.css stub before shadcn init**
- **Found during:** Task 6 (Shadcn init)
- **Issue:** shadcn CLI requires a Tailwind CSS file to exist to detect v4 — empty `src/styles/globals.css` must exist with `@import "tailwindcss"` before running init
- **Fix:** Created stub globals.css with `@import "tailwindcss"` before running shadcn init
- **Files modified:** dashboard/src/styles/globals.css
- **Verification:** shadcn init succeeded with "Validating Tailwind CSS. Found v4. OK"
- **Committed in:** 94e669f (Task 6 commit)

## Issues Encountered
